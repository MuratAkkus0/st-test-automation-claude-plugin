---
name: phase2-pre-clean-partner-domain
description: Phase 2 must pre-clean the partner domain (cookies + localStorage + sessionStorage) and reload BEFORE snapshot/consent/storage-check. The post-consent "never reload before storage check" rule does NOT cover this pre-consent reload.
type: lesson
---

Phase 2's very first action (Step 2.0) is to clear the partner domain's cookies, localStorage, and sessionStorage, then reload the same URL (the `moeclid` query param is preserved across the reload). Only after that does the existing snapshot / accept-consent / dismiss-popups / storage-check flow run. Phase 0 already clears the portal domain, but Phase 0 doesn't know the partner domain yet — Phase 2 is the first point in the test where the partner domain is known.

**Why:** Without this step, on a re-run against the same partner the storage check is ambiguous: a stored `moeclid` cookie or `MOEBEL_CLICKOUT_ID` from yesterday's run looks identical to one freshly written by today's Base Tag, so a broken integration can pass the test by inheriting leftover state. The same problem applies to a leftover `CookieConsent` cookie — it skips the consent banner entirely, which means Step 2.2 records "no banner found" and we never exercise the consent-callback code path that real first-visit users go through. The pre-clean + reload guarantees a true first-visit state for every Phase 2 run, regardless of test history on the same partner.

**How to apply:** In Phase 2 Step 2.0, run the same `document.cookie` / `localStorage.clear()` / `sessionStorage.clear()` JS that Phase 0 uses on the portal domain (one expression, executed via `evaluate_script` on the current page handle), then call `navigate_page(page=newPageId, action="reload")`. Do this BEFORE the first snapshot, BEFORE consent acceptance, BEFORE any storage read. The post-consent "never reload before storage check" rule is a different rule with a different scope — it forbids reloads between consent acceptance (Step 2.2/2.3) and the storage check (Step 2.5/2.6) because such a reload would let the Base Tag fire on the reloaded `page_view` even when it would never fire for a real user. The pre-consent pre-clean reload happens before consent, so the two rules coexist cleanly. Both the Phase 2 SKILL and the `st-storage-inspector` agent description must keep the pre-consent vs post-consent distinction explicit so a future reader doesn't delete the pre-clean step thinking it violates the older rule.
