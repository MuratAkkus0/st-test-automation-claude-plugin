### Notes on consent state

The Plenty-Shop cookie `plenty-shop-cookie` showed `consent:true` already on first paint of the partner page. Because Phase 0 only clears the portal's cookies (the partner domain isn't known yet at that stage), the partner site can carry residual consent state from a previous browser session. In this run, that residual state means the consent banner did not need to appear at all on the test visit. The Base Part still correctly fired and stored the clickId, which is the behaviour we want: as a returning visitor with consent on file, the tracking script ran on `page_view` and read the moeclid out of the URL on the same page load.

If a future test needs to verify the consent-gated path (banner shown → accept → tag fires), clear cookies on `naturwohnen.de` directly before navigating.
